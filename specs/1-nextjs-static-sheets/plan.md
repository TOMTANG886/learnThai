# Implementation Plan: Static Next.js site from Google Sheets

**Branch**: `1-nextjs-static-sheets` | **Date**: 2026-03-07 | **Spec**: ../1-nextjs-static-sheets/spec.md
**Input**: Feature specification from `/specs/1-nextjs-static-sheets/spec.md`

## Summary

Read Google Sheets as the single source of truth, generate pre-rendered Next.js pages for each published row, include audio external-asset URLs from a designated column, and produce a reproducible static output plus a build manifest.
Also generate a per-worksheet table page at `/worksheets/<worksheet-slug>` that lists all rows from that worksheet as a table and links each row to its per-row page.

## Technical Context

**Language/Version**: Node.js 18+ (Next.js 13+)  
**Primary Dependencies**: Next.js, `googleapis` (Sheets), `gray-matter`, `remark`/`remark-html`, `node-fetch` or built-in fetch, optional `esbuild` for asset tasks, `firebase-tools` for deploys.  
**Storage**: Google Sheets (single source of truth).  
**Testing**: Vitest for unit tests and contract tests for sheet schema validation.  
**Target Platform**: Static hosting (Firebase Hosting recommended — SEE research.md).  
**Project Type**: Static site generator / build script integrated with Next.js.  
**Performance Goals**: Build completes within 2 minutes for ~100 posts on a typical dev machine.  
**Constraints**: No runtime server endpoints in production; all data consumption at build time.  
**Scale/Scope**: Single-author blog, expected <5k posts over lifetime; pipeline designed for content volume up to thousands of rows.

## Constitution Check

- Principle I (Static-First): Compliant — output is pre-rendered static pages.  
- Principle II (Sheets SOT): Compliant — content originates from Google Sheets exclusively.  
- Principle III (Isolated Services): Compliant — translation/TTS are out-of-scope; audio is an external-asset column and not generated by the pipeline in this feature. Any future translation/TTS must follow service module pattern.  
- Principle IV (Idempotent Pipeline): Compliant — plan includes build manifest and content hashing to ensure idempotency.  
- Principle V (Simplicity): Compliant — single pipeline script preferred.

Gates: No constitution violations found for this feature. Remaining operational choices (deployment target and auth method) are recorded as decisions in `research.md`.

```text

├── package.json
├── src/
│   ├── lib/
```

**Structure Decision**: Single project (Next.js app) with a `scripts/build-from-sheets.js` orchestration entrypoint used by `npm run build:sheets` before `next build` / `next export`.

Additional output: `pages/worksheets/[worksheet].tsx` (or pre-rendered HTML) generated from worksheet metadata; `public/worksheets/<worksheet-slug>/index.html` for static export.
